# 로컬 검증 로그

## 검증 목적

CLI helper가 small model에게 줄 수 있는 핵심 이점은 속도 자체보다 output shape다. 이 검증은 helper CLI가 broad shell command보다 더 작고 구조화된 evidence packet을 만들 수 있는지 확인했다.

## 환경

- Shell: `zsh`
- `ripgrep 14.1.1`
- `fd 10.2.0`
- `jq 1.7.1`
- Mike Farah `yq v4.47.2`
- `ast-grep`: 미설치
- `mdq`: 미설치

## Fixture

검증 fixture는 원본 ultraresearch 작업 디렉터리에 있다.

```text
.omo/ultraresearch/20260621-opencode-small-model-tools/fixtures/
```

포함 파일:

- `src/api.ts`
- `src/config.ts`
- `package.json`
- `workflow.yaml`
- `README.md`

## 실행 결과

### `fd`

명령:

```sh
fd --type f . .omo/ultraresearch/20260621-opencode-small-model-tools/fixtures
```

결과:

- 5개 fixture file을 clean path list로 반환했다.
- `find`보다 model이 parsing하기 쉬운 bounded path list를 만들었다.

### `fd` extension filter

명령:

```sh
fd --type f --extension ts . .omo/ultraresearch/20260621-opencode-small-model-tools/fixtures/src
```

결과:

- `api.ts`, `config.ts`만 반환했다.

### `rg --json`

명령:

```sh
rg --json --line-number --column --no-heading 'console\.log' .omo/ultraresearch/20260621-opencode-small-model-tools/fixtures/src
```

결과:

- `begin`, `match`, `end`, `summary` NDJSON event를 반환했다.
- match event에는 path, line text, line number, absolute offset, submatch가 포함됐다.
- plain `grep -R -n`보다 model이 안정적으로 후처리할 수 있는 구조다.

### `grep` 비교

명령:

```sh
grep -R -n 'console\.log' .omo/ultraresearch/20260621-opencode-small-model-tools/fixtures/src
```

결과:

- 사람이 읽기에는 충분했지만, JSON field, offset, submatch, summary가 없다.
- small model이 다시 text parsing을 해야 한다.

### `jq`

명령:

```sh
jq -c '{name, scripts, dependencyNames: (.dependencies | keys)}' .omo/ultraresearch/20260621-opencode-small-model-tools/fixtures/package.json
```

결과:

```json
{"name":"small-model-tool-fixture","scripts":{"test":"node --test","lint":"eslint ."},"dependencyNames":["zod"]}
```

필요한 field만 compact JSON으로 반환했다.

### `yq`

명령:

```sh
yq -o=json '.jobs.test.steps' .omo/ultraresearch/20260621-opencode-small-model-tools/fixtures/workflow.yaml
```

결과:

```json
[
  {
    "uses": "actions/checkout@v4"
  },
  {
    "run": "npm test"
  }
]
```

YAML path selection을 JSON으로 변환해 downstream model/tool이 바로 소비할 수 있게 했다.

## 미검증 항목

- `ast-grep`: 로컬 미설치로 실행 검증 실패. 문서와 source 기반으로만 판단했다.
- `mdq`: 로컬 미설치로 실행 검증 실패. `yshavit/mdq` 문서 기반으로만 판단했다.
- 실제 Qwen3.6/OpenCode end-to-end run은 수행하지 않았다.

## 검증 결론

`rg`, `fd`, `jq`, `yq`는 small model의 parsing 부담을 실제로 줄인다. 특히 `rg --json`과 `jq/yq` projection은 “파일 전체를 모델에 넣는 방식”보다 훨씬 작은 evidence packet을 만든다. 반면 이 검증은 model success rate 자체를 증명하지 않는다.
