# 리서치 추적

## 질문

Qwen3.6급 small model을 OpenCode에 연결할 때 `ast-grep`, `ripgrep`, `fd`, `yq`, `jq`, `mdq` 같은 CLI를 사용해 모델을 보조하고 기본 command를 잘 대체할 수 있는가?

## 조사 축

1. OpenCode tool/custom tool/MCP/permission surface.
2. Qwen3.6/Qwen3-Coder small model의 agentic coding evidence.
3. `ast-grep`의 AST search/rewrite와 MCP 가능성.
4. `rg`/`fd`의 path/text retrieval 적합성.
5. `jq`/`yq`/`mdq`의 structured data/Markdown extraction 적합성.
6. Tiny-Chu/Tinker.Gen으로 이전 가능한 구조.
7. 로컬 실행 검증.

## Wave 1

병렬 조사:

- OpenCode 공식 문서와 현재 repo snippet.
- OpenCode repo deep dive.
- Qwen3.6/Qwen3-Coder model card, blog, 논문.
- ast-grep 문서, JSON mode, rewrite, MCP.
- ripgrep/fd docs와 실제 CLI option.
- jq/yq/mdq docs.

주요 결과:

- OpenCode는 `ast-grep`, `fd`, `jq`, `yq`, `mdq`를 native built-in으로 제공하지 않는다.
- OpenCode는 custom tools와 MCP server를 통해 helper tool을 확장할 수 있다.
- `rg`는 OpenCode built-in `grep` 경로와 이미 일부 겹친다.
- Qwen 계열은 agentic coding을 목표로 하지만 tool wrapper가 추론력을 대체하지 않는다.
- `mdq`는 이름 충돌이 있어 exact implementation pin이 필요했다.

## Dead end

한 worker가 `opencode-ai/opencode` Go 구현을 조사했다. 그러나 현재 공식 OpenCode 경로는 `anomalyco/opencode` / `sst/opencode` 계열이므로, 해당 Go 구현은 current implementation 근거로 쓰지 않았다.

유지한 교훈:

- tool registry
- MCP extension
- permission gating
- read-before-write
- diff-before-apply

하지만 current implementation claim에는 사용하지 않았다.

## Wave 2

확장 조사:

- `mdq` exact implementation pin.
- `ast-grep-mcp` 확인.
- 현재 OpenCode custom tool 예시 확인.
- real-world `rg --json` wrapper pattern 확인.
- 로컬 output-shape 검증.

결과:

- `mdq`는 `yshavit/mdq`를 기준 구현으로 pin했다.
- current OpenCode plugin example은 `tool.schema` 기반 typed custom tool 구현을 보여준다.
- `ast-grep-mcp`는 AI assistant 대상 structural code search MCP server로 확인됐다.
- 여러 OSS tool이 `rg --json`을 parse해 normalized search result로 쓰는 pattern을 보였다.
- 로컬에서 `rg`, `fd`, `jq`, `yq` output shape를 검증했다.

## Wave 3 수렴

새로운 research lead는 남지 않았다. 남은 것은 구현 선택이다.

- `.opencode/tools`로 만들 것인가?
- local MCP server로 만들 것인가?
- `ast-grep`/`mdq`를 필수 bootstrap dependency로 둘 것인가?
- Tiny-Chu에 넣을 것인가, 별도 OpenCode helper package로 둘 것인가?

## 최종 판단

CLI helper는 small model의 context와 parsing 부담을 줄이는 데 유효하다. 그러나 효과를 내려면 broad shell 자유도가 아니라 typed helper tool, schema, output cap, permission 정책으로 제한해야 한다.
